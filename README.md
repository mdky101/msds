# 이거 위험한가요?

화학제품 라벨을 폰으로 찍으면 GHS 그림문자를 읽어 위험성을 알려주고, 국문 MSDS를 찾아주는 웹앱.

## 무엇을 하나

1. **찍으면 즉답** — 라벨의 GHS 그림문자·신호어로 위험도를 판정한다. CAS를 못 읽어도 동작한다.
2. **국문 MSDS 찾기** — 제품명으로 두 곳을 동시에 검색한다.
   - 제조사 공식 자료(네이버 웹 검색) — 상표명 제품(WD-40 등)은 여기서 찾힌다
   - 안전보건공단 정식 자료(KOSHA 오픈API) — 법정 자료. 물질명·CAS로 찾을 때 걸린다
3. **요약 보기** — 법정 16항 중 현장에서 쓰는 5항(유해성·응급조치·화재·취급저장·보호구)을 발췌한다.
4. **PDF 구성성분** — 검색된 MSDS PDF를 열어 3항 구성성분 표를 옮겨 적는다.

## 설계 원칙

**위험도 판정에 AI가 개입하지 않는다.** 비전 모델은 "라벨에 어떤 그림문자가 보이는가"만 답하고,
그것이 얼마나 위험한지는 `src/lib/ghs.ts`의 고정 규칙이 정한다. 모델이 아는 지식으로 빈칸을
채우면 그건 판독이 아니라 기억이고, 틀렸을 때 사용자가 검증할 방법이 없다.

**"안전" 등급은 없다.** 최저 등급은 "정보 부족"까지다. 그림문자가 안 보이는 건 안전하다는 뜻이
아니라 라벨이 가려졌거나 사진이 흐릴 수 있다는 뜻이다.

**요약은 규칙 기반이다.** MSDS 16항은 법정 정형 서식이라 LLM 요약이 필요 없다. 항목을 고를 뿐
문장을 바꾸지 않는다. 안전정보에서 환각은 사람이 다친다.

**교차검증.** 사진에서 읽은 CAS와 PDF의 CAS가 어긋나면 경고한다. 사진의 그림문자와 공단 공식
그림문자가 다르면 알린다(어느 쪽이 틀렸다고 단정하지는 않는다).

## 로컬 실행

```bash
npm install
cp .env.example .env.local   # 키를 채운다
npm run dev
```

| API | 한도 | 앱의 소모 |
| --- | --- | --- |
| **Gemini** | **20회/일** (모델별) | 사진 1장 = 1회, PDF 성분 = 1회 |
| KOSHA | 1,000회/일 | 검색 1회당 최대 4회, 요약 1회당 5회 |

병목은 Gemini다. **하루 약 20장**이 상한이며, 이는 문서에 적힌 값(1,500)이 아니라 429 응답의
`quotaValue`에서 확인한 실제 값이다. 더 필요하면 결제 등록(Tier 1)이 필요하다.

## 한계

- **상표명 제품은 공단 자료에 없다.** 공단은 물질 DB다. 제조사 웹 검색이 이를 메운다.
- **제품 변형을 자동으로 특정하지 못한다.** 같은 브랜드도 종류마다 MSDS가 다르다(예: WD-40
  Multi-Use / Specialist Degreaser). 그래서 PDF의 제품명·개정일을 항상 함께 보여준다.
- **웹의 제조사 MSDS는 대체로 구버전이다.** 실측 개정일이 2016·2018·2022년이었다.
- **한글 회사명과 영문 도메인은 대조할 수 없다.** 공식이어도 "출처 미확인"으로 남는다.
- **스캔 이미지 PDF는 미검증이다.** 못 읽으면 추측하지 않고 물러선다.

## 면책

이 앱의 판정은 참고용이다. 법적 효력이 있는 자료는 해당 제품의 원본 MSDS이며, 취급 전 반드시
원본을 확인해야 한다.
